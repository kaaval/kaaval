# Getting help with Kaaval

Thanks for using Kaaval. Here's the fastest way to get what you need.

## Where to go

| I want to… | Go to |
|---|---|
| Ask a question, share a scan result, or float an idea | [Discussions](https://github.com/kaaval/kaaval/discussions) |
| Report a bug | [Open a bug report](https://github.com/kaaval/kaaval/issues/new/choose) |
| Request a detection rule or feature | [Open a rule/feature request](https://github.com/kaaval/kaaval/issues/new/choose) |
| Report a **security vulnerability** | **Do not open a public issue** — follow [SECURITY.md](SECURITY.md) |
| Contribute code | [CONTRIBUTING.md](CONTRIBUTING.md) |

Before opening an issue, a quick search of existing
[issues](https://github.com/kaaval/kaaval/issues) and
[Discussions](https://github.com/kaaval/kaaval/discussions) often turns up an
answer already.

## Response expectations

Kaaval is maintained by a small team on nights and weekends, across time zones.
We read everything, and we care — but we're not a 24/7 support desk. In
practice, expect a response within **a few days**, sometimes sooner. Security
reports are triaged first.

If something's been quiet for over a week, a friendly bump on the thread is
completely welcome — it's not nagging, it just helps things resurface.

Nothing reaches a release without passing CI and review, so the project stays
safe to move at a sustainable pace. Thanks for your patience — it's what keeps
open source healthy for the long run.
